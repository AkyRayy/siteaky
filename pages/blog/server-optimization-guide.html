Оптимизация JVM параметров

Правильная настройка JVM (Java Virtual Machine) — это основа производительности любого Minecraft сервера. Вот оптимальные параметры для разных ситуаций:

🚀 Оптимальные JVM параметры для серверов 1-4 ГБ RAM:

Bash 
java -Xms2G -Xmx2G \
-XX:+UseG1GC \
-XX:+ParallelRefProcEnabled \
-XX:MaxGCPauseMillis=200 \
-XX:+UnlockExperimentalVMOptions \
-XX:+DisableExplicitGC \
-XX:+AlwaysPreTouch \
-XX:G1NewSizePercent=30 \
-XX:G1MaxNewSizePercent=40 \
-XX:G1HeapRegionSize=8M \
-XX:G1ReservePercent=20 \
-XX:G1HeapWastePercent=5 \
-XX:G1MixedGCCountTarget=4 \
-XX:InitiatingHeapOccupancyPercent=15 \
-XX:G1MixedGCLiveThresholdPercent=90 \
-XX:G1RSetUpdatingPauseTimePercent=5 \
-XX:SurvivorRatio=32 \
-XX:+PerfDisableSharedMem \
-XX:MaxTenuringThreshold=1 \
-Dusing.aikars.flags=https://mcflags.emc.gs \
-Daikars.new.flags=true \
-jar server.jar nogui

💪 Для серверов 8-16 ГБ RAM:

Bash 
java -Xms6G -Xmx6G \
-XX:+UseG1GC \
-XX:+ParallelRefProcEnabled \
-XX:MaxGCPauseMillis=150 \
-XX:+UnlockExperimentalVMOptions \
-XX:+DisableExplicitGC \
-XX:+AlwaysPreTouch \
-XX:G1NewSizePercent=40 \
-XX:G1MaxNewSizePercent=50 \
-XX:G1HeapRegionSize=16M \
-XX:G1ReservePercent=15 \
-XX:G1HeapWastePercent=5 \
-XX:G1MixedGCCountTarget=8 \
-XX:InitiatingHeapOccupancyPercent=20 \
-XX:G1MixedGCLiveThresholdPercent=85 \
-XX:G1RSetUpdatingPauseTimePercent=8 \
-XX:SurvivorRatio=16 \
-XX:MaxTenuringThreshold=1 \
-Dusing.aikars.flags=https://mcflags.emc.gs \
-XX:G1MixedGCCountTarget=8 \
-jar server.jar nogui

📋 Пояснение ключевых параметров:

  • -Xms/-Xmx — начальный и максимальный размер кучи
  • -XX:+UseG1GC — сборщик мусора G1 (оптимальный для Minecraft)
  • -XX:MaxGCPauseMillis — максимальная пауза GC
  • -XX:InitiatingHeapOccupancyPercent — порог запуска GC
  • -XX:+AlwaysPreTouch — предварительное выделение памяти

Выбор серверного ядра

Выбор правильного серверного ядра критически важен для производительности. Рассмотрим самые популярные варианты:

Ядро Производительность Совместимость Рекомендация
Paper Высокая Отличная 🥇 Лучшая рекомендация
Purpur Очень высокая Хорошая 🥈 Для максимальной производительности
Folia Экстремальная Средняя 🥉 Для крупных серверов
Spigot Средняя Отличная ⚠️ Устаревший вариант

🎯 Наша рекомендация: PaperMC

PaperMC — это золотой стандарт для Minecraft серверов. Он предлагает отличную производительность, полную совместимость с плагинами и активную разработку.

Bash 
# Установка PaperMC
wget https://api.papermc.io/v2/projects/paper/versions/1.21.4/builds/545/downloads/paper-1.21.4-545.jar
mv paper-1.21.4-545.jar server.jar

# Запуск с оптимизированными параметрами
./start.sh

Мониторинг производительности

Регулярный мониторинг помогает выявлять проблемы производительности на ранней стадии. Рекомендуем использовать следующие инструменты:

📊 Spark Profiler

Мощный профилировщик для анализа производительности сервера

Command 
/spark profiler start --timeout 60

📈 Chunky

Инструмент для предгенерации мира и анализа чанков

Command 
/chunky start --circle 5000

🔍 Clarity

Мониторинг производительности в реальном времени

Command 
/clarity monitor

Продвинутые техники оптимизации

1. Предгенерация мира

Предгенерация мира предотвращает лаги при исследовании новых территорий игроками.

Bash 
# Установка Chunky
wget https://chunky.llbit.se/Chunky-2.4.29.jar

# Предгенерация мира радиусом 5000 блоков
java -jar Chunky-2.4.29.jar -w world -c world -r 5000

2. Оптимизация плагинов

Некоторые плагины могут значительно влиять на производительность. Регулярно проверяйте их с помощью Spark.

  • Используйте асинхронные операции для тяжёлой работы
  • Оптимизируйте частоту тиков плагинов
  • Избегайте избыточных проверок в циклах
  • Используйте кэширование для часто запрашиваемых данных

3. Настройка Redstone

YAML 
# Ограничение Redstone в paper.yml
settings:
  redstone:
    physics:
      redstone-nature:
        # Отключение некоторых Redstone механик
        disable-legacy-update: true
        disable-update-on-block-break: true
        disable-update-on-place: true
    # Уменьшение частоты проверок
    repeat-delay-max: 2
    repeat-delay-min: 1

Заключение

Оптимизация Minecraft сервера — это непрерывный процесс, требующий регулярного мониторинга и настройки. Следуя этому руководству, вы сможете добиться значительного улучшения производительности и обеспечить плавную работу сервера даже при большом количестве игроков.

🎯 Ключевые выводы:

  • Используйте PaperMC с оптимизированными JVM параметрами
  • Настройте view-distance в соответствии с мощностью сервера
  • Регулярно мониторьте производительность с помощью Spark
  • Предгенерируйте мир для предотвращения лагов
  • Оптимизируйте конфигурационные файлы под ваши нужды
  • Следите за производительностью плагинов
