Асинхронное программирование в плагинах Minecraft

Как использовать CompletableFuture и Bukkit Scheduler для создания отзывчивых плагинов без лагов сервера

👨‍💻
AkyDev Team
Разработчики плагинов
⏱️ 8 мин чтения 👁️ 1.2K просмотров

В современном Minecraft сервере производительность — это ключ к успеху. Когда плагины выполняют тяжёлые операции в основном потоке, это приводит к лагам, фризам и плохому пользовательскому опыту. В этой статье мы разберём, как использовать асинхронное программирование для создания высокопроизводительных плагинов.

Что такое асинхронное программирование?

Асинхронное программирование — это парадигма, позволяющая выполнять операции без блокировки основного потока. В Minecraft это особенно важно, так как основной поток отвечает за обработку игровых событий, физику и рендеринг.

Java 
// Синхронный код (плохо)
public void onPlayerJoin(PlayerJoinEvent event) {
    // Блокирует основной поток!
    String uuid = fetchUUIDFromDatabase(event.getPlayer().getName());
    event.getPlayer().sendMessage("Ваш UUID: " + uuid);
}

// Асинхронный код (хорошо)
public void onPlayerJoin(PlayerJoinEvent event) {
    Bukkit.getScheduler().runTaskAsynchronously(plugin, () -> {
        String uuid = fetchUUIDFromDatabase(event.getPlayer().getName());
        
        // Возвращаем в основной поток для безопасных операций
        Bukkit.getScheduler().runTask(plugin, () -> {
            event.getPlayer().sendMessage("Ваш UUID: " + uuid);
        });
    });
}

Bukkit Scheduler основы

Bukkit Scheduler предоставляет несколько методов для асинхронного выполнения задач:

📋 Типы задач Bukkit Scheduler:

  • runTask() — синхронное выполнение в следующем тике
  • runTaskAsynchronously() — асинхронное выполнение
  • runTaskLater() — отложенное синхронное выполнение
  • runTaskLaterAsynchronously() — отложенное асинхронное выполнение
  • runTaskTimer() — повторяющееся синхронное выполнение
  • runTaskTimerAsynchronously() — повторяющееся асинхронное выполнение
Java 
// Пример асинхронной задачи с задержкой
public void scheduleAsyncTask() {
    Bukkit.getScheduler().runTaskLaterAsynchronously(plugin, () -> {
        // Тяжёлая операция (запрос к API, база данных и т.д.)
        processData();
        
        // Возвращаем в основной поток
        Bukkit.getScheduler().runTask(plugin, () -> {
            // Безопасная операция с Bukkit API
            Bukkit.broadcastMessage("Обработка завершена!");
        });
    }, 20L); // Задержка в 1 секунду (20 тиков)
}

CompletableFuture в Minecraft

CompletableFuture — мощный инструмент Java 8+ для асинхронного программирования. Он позволяет создавать цепочки асинхронных операций:

Java 
public CompletableFuture<PlayerData> loadPlayerDataAsync(UUID uuid) {
    return CompletableFuture
        .supplyAsync(() -> loadFromDatabase(uuid)) // Асинхронная загрузка
        .thenApply(data -> processData(data))       // Обработка данных
        .thenApply(data -> cacheData(data))          // Кэширование
        .whenComplete((data, throwable) -> {
            if (throwable != null) {
                logger.error("Ошибка загрузки данных игрока", throwable);
            } else {
                // Возвращаем в основной поток для использования
                Bukkit.getScheduler().runTask(plugin, () -> {
                    applyDataToPlayer(uuid, data);
                });
            }
        });
}

Лучшие практики

✅ Правильно: Используйте пул потоков

Java 
// Создаём свой пул потоков
private ExecutorService asyncPool = Executors.newFixedThreadPool(4);

public void processAsync() {
    asyncPool.submit(() -> {
        // Тяжёлая операция
        heavyOperation();
    });
}

❌ Неправильно: Блокировка основного потока

Java 
// Никогда так не делайте!
public void onCommand(CommandSender sender, Command cmd, String label, String[] args) {
    // Это заблокирует весь сервер!
    String result = heavyWebRequest(); 
    sender.sendMessage(result);
}

Практические примеры

Пример 1: Асинхронная загрузка данных

Java 
public class AsyncDataManager {
    private final Map<UUID, PlayerData> cache = new ConcurrentHashMap<>();
    
    public void loadPlayerData(UUID uuid, Consumer<PlayerData> callback) {
        // Проверяем кэш
        PlayerData cached = cache.get(uuid);
        if (cached != null) {
            callback.accept(cached);
            return;
        }
        
        // Асинхронная загрузка
        Bukkit.getScheduler().runTaskAsynchronously(plugin, () -> {
            try {
                PlayerData data = database.loadPlayerData(uuid);
                cache.put(uuid, data);
                
                // Возвращаем в основной поток
                Bukkit.getScheduler().runTask(plugin, () -> {
                    callback.accept(data);
                });
            } catch (Exception e) {
                plugin.getLogger().error("Ошибка загрузки данных", e);
            }
        });
    }
}

Пример 2: Асинхронные API запросы

Java 
public class AsyncAPIManager {
    private final HttpClient httpClient = HttpClient.newHttpClient();
    
    public CompletableFuture<String> makeApiRequest(String url) {
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(url))
            .GET()
            .build();
            
        return httpClient
            .sendAsync(request, HttpResponse.BodyHandlers.ofString())
            .thenApply(HttpResponse::body)
            .exceptionally(throwable -> {
                logger.error("API запрос не удался", throwable);
                return null;
            });
    }
    
    public void checkPlayerStatus(Player player) {
        makeApiRequest("https://api.example.com/status/" + player.getUniqueId())
            .thenAccept(response -> {
                Bukkit.getScheduler().runTask(plugin, () -> {
                    if (response != null) {
                        player.sendMessage("Статус: " + response);
                    }
                });
            });
    }
}

Заключение

Асинхронное программирование — это незаменимый инструмент для создания высокопроизводительных Minecraft плагинов. Правильное использование Bukkit Scheduler и CompletableFuture позволяет избежать лагов и обеспечить плавную работу сервера даже при выполнении сложных операций.

🎯 Ключевые выводы:

  • Никогда не выполняйте тяжёлые операции в основном потоке
  • Используйте Bukkit Scheduler для асинхронных задач
  • CompletableFuture отлично подходит для цепочек операций
  • Всегда возвращайтесь в основной поток для работы с Bukkit API
  • Создавайте пулы потоков для лучшей производительности
